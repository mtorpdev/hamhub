# Full App Internationalization Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add complete English/Danish UI language support across HamHub with English default, browser detection, local override, and authenticated profile persistence.

**Architecture:** Backend stores `PreferredLanguage` on `ApplicationUser` and exposes it through existing `UserDto`/`UpdateUserDto`. Frontend owns all UI strings through a lightweight internal i18n layer: `LanguageProvider`, `useT()`, typed dictionaries, browser detection, localStorage fallback, and profile sync. Existing routes stay unchanged; this is text localization, not URL localization.

**Tech Stack:** ASP.NET Core 8, EF Core/Npgsql schema guards, Next.js 16 App Router, React context/hooks, TypeScript translation dictionaries, existing HamHub API client.

## Global Constraints

- Supported languages are exactly `en` and `da`.
- Default language is English.
- Browser language selects Danish only for `da`/`da-*`; all other browsers use English.
- Authenticated users can save preferred language on their profile.
- Anonymous users can override language via localStorage.
- UI language must cover pages, nested detail pages, popups, modals, toasts, confirm dialogs, empty states, validation/error helper text, tabs, buttons, navigation, and reusable components.
- User-generated content, call signs, logbook data, forum posts, messages, article bodies, external API responses, and radio abbreviations are not machine-translated.
- Keep route paths unchanged for now.
- Permanent build rule: every future feature or UI change must add/update both English and Danish translation keys at the same time, including subpages, popups, modals, toasts, confirmation prompts, notification text, empty states, validation text, and reusable component labels.
- Permanent verification rule: future UI verification must include at least one English and one Danish smoke check for the changed flow.
